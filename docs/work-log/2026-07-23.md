## 오늘의 요약

오늘은 SilverLink AI를 Google Play Store에 제출하기 위한 출시 준비 작업 전체를 진행했다. 개인정보처리방침 HTML 페이지를 직접 작성해 GitHub Pages에 배포했고, Play Console 스토어 등록정보(앱 설명·그래픽 이미지·스크린샷·카테고리)를 모두 채웠다. 그래픽 자료는 HTML Canvas + Edge 헤드리스 브라우저를 이용해 코드로 직접 렌더링해 PNG 파일로 저장했다. - 스크린샷 3종(1080×1920): 홈 대시보드 / 안전 알림 / AI 분석 결과 모의 화면
- 저장: `silverlink-mobile/assets/[그래픽이미지]_1024x500_PlayStore배너.png` 외 3종
- 앱 아이콘: 기존 `assets/icon.png` (512×512) 그대로 사용

**`app.json` 수정:**
- `expo.updates.runtimeVersion` → `expo.runtimeVersion`으로 위치 이동 (위치 오류 수정)
- bare workflow 제약으로 `{ "policy": "appVersion" }` → 문자열 `"1.0.56"` 변경

**`android/app/build.gradle` 수정:**
- `versionCode 57 → 58` (57은 Play Console에 이미 업로드된 이력 있음)
- `versionName "1.0.56"` 유지 (기능 변경 없으므로 사용자 표시 버전 동일)

### 문제와 해결 과정

1. **runtimeVersion 위치 오류**: `expo.updates.runtimeVersion`에 넣었더니 EAS 빌드 시 "missing from your Expo app config" 에러 발생 → `expo.runtimeVersion`(최상단)으로 이동해 해결
2. **bare workflow policy 미지원**: `{ "policy": "appVersion" }`은 managed workflow 전용 기능 → `"1.0.56"` 문자열로 수동 지정
3. **versionCode 57 중복**: Play Console에 이전 업로드 이력 있어 "57 버전 코드는 이미 사용됨" 오류 → 58로 증가
4. **EAS 클라우드 한도 초과**: 무료 플랜 월 빌드 횟수 소진 (8월 1일 리셋) → EAS 로컬 빌드 시도
5. **EAS 로컬 빌드 Windows 미지원**: `--local` 옵션이 macOS/Linux 전용 → WSL2 확인했으나 Gradle 직접 빌드로 해결
6. **Gradle 빌드 성공**: `keystore.properties` + `android/app/release.keystore` 이미 존재 → `gradlew.bat bundleRelease` 1분 22초 완료, 35MB AAB 생성

### 기술적 의사결정

- **그래픽 자료 생성 방식**: Figma/포토샵 없이 HTML Canvas + Edge 헤드리스 브라우저로 프로그래밍적으로 생성. 외부 도구 의존 없이 코드만으로 완결.
- **runtimeVersion 관리**: bare workflow에서는 문자열 수동 지정 필수. 앞으로 버전 올릴 때 4곳 동기화 필요: `versionCode`, `versionName`(build.gradle), `APP_VERSION`(settings.tsx), `runtimeVersion`(app.json)
- **빌드 경로**: EAS 클라우드 → EAS 로컬(Windows 불가) → Gradle 직접 빌드. release.keystore가 로컬에 있어 서명된 AAB 생성 가능했음.
