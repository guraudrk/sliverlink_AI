## 세션 5: EAS 빌드 우회 + Supabase 키 수정 + Google OAuth 추가

### 요청과 목표

- EAS 무료 티어 구독 제한으로 인해 AAB를 로컬에서 빌드하는 방법 탐색
- 앱 로그인 시 "invalid api key" 오류 근본 원인 파악 및 해결
- 로그인 화면에 Google 소셜 로그인 기능 추가

### 구현 및 변경 내용

| 항목 | 내용 |
|---|---|
| EAS 빌드 완료 확인 | 큐 대기 중이던 bdca25df 빌드(versionCode 3, v1.0.2) 완료 — AAB 다운로드 링크 확보 |
| Supabase anon key 교정 | EAS 대시보드 production 환경변수의 `EXPO_PUBLIC_SUPABASE_ANON_KEY` 값이 잘못 등록되어 있었음. 사용자가 Supabase 대시보드에서 현재 키 확인 후 EAS 환경변수 갱신 |
| `expo-auth-session`, `expo-crypto` 설치 | Google OAuth PKCE 플로우 구현에 필요한 패키지 추가 |
| `lib/supabase.ts` 수정 | `flowType: "pkce"` 추가 — OAuth 코드 교환(PKCE) 활성화 |
| `app/(auth)/login.tsx` 수정 | Google 로그인 버튼 추가 (`WebBrowser.openAuthSessionAsync` + `supabase.auth.exchangeCodeForSession`) |
| EAS 빌드 재제출 | versionCode 4 / v1.0.3, 올바른 Supabase 키 + Google OAuth 포함한 새 빌드 제출 (bypdbikw7) |

### 문제와 해결 과정

- **`eas build --local` Windows 미지원**: 에러 메시지 `Unsupported platform, macOS or Linux is required`. Windows에서는 실행 불가. Gradle 직접 빌드(`gradlew.bat bundleRelease`)를 시도했으나 빌드가 오래 걸리고 `expo-eas-client` Kotlin 컴파일 실패 감지됨
- **로컬 Gradle 빌드 `.gradle` 잠금 충돌**: Gradle 프로세스가 아직 실행 중인 상태에서 EAS 업로드를 시도하자 `checksums.lock` 파일 잠금 충돌 발생. Java 프로세스 전체 종료 후 재시도
- **versionCode 3 AAB 활용 결정**: 이미 완료된 EAS 클라우드 빌드 AAB를 사용하는 것이 서명 키 일치 측면에서 가장 안전. 단, Supabase 키 오류가 있어 해당 빌드는 Play Store 업로드용으로는 부적합
- **Supabase `silverlink://auth/callback` 이미 등록됨**: URL Configuration에 이미 등록되어 있었고 Google Provider도 활성화 상태 — 추가 설정 불필요

### 기술적 의사결정

- **로컬 Gradle vs EAS 클라우드**: Windows 환경의 EAS 로컬 빌드 제약으로 결국 EAS 클라우드 빌드로 회귀. 올바른 Supabase 키 설정 후 클라우드 빌드가 더 신뢰성 높은 방법
- **Google OAuth 구현 방식**: 네이티브 SDK(`@react-native-google-signin`) 대신 `expo-auth-session` + `WebBrowser` 방식 선택. 추가 네이티브 모듈 없이 구현 가능하며 Expo 관리형 워크플로우와 호환성 우수
- **Naver OAuth 보류**: Supabase 미지원, 네이티브 SDK 필요, 네이버 개발자센터 검토 기간 등 복잡도가 높아 별도 작업으로 분리

### 변경 파일

- `silverlink-mobile/lib/supabase.ts`: `flowType: "pkce"` 추가
- `silverlink-mobile/app/(auth)/login.tsx`: Google 로그인 버튼 및 OAuth 핸들러 추가
- `silverlink-mobile/package.json`: `expo-auth-session`, `expo-crypto` 추가

### 미해결 사항 및 다음 작업

- EAS 빌드(bypdbikw7, versionCode 4) 완료 대기 중 — 완료 후 AAB 다운로드 및 Play Console 업로드
- Google Cloud Console에서 OAuth 2.0 클라이언트 ID 생성 + Supabase Google Provider에 등록 필요 (현재 Provider는 켜져 있으나 Client ID/Secret 미확인)
- Play Console 업로드 후 내부 테스터 실기기 로그인 테스트 필요
