## 🎯 북극성 목표 (AI 에이전트에게 내리는 최상위 명령)

> **월 매출 1,000만 원을 최대한 빠르게 실현한다.**
> 방법론은 "월 매출 1,000만 원 이상을 달성한 AI 1인 기업의 검증된 방법론"을 따른다.

### AI 1인 기업 검증 방법론 원칙

1. **제품보다 고객 먼저** — 코드 한 줄 쓰기 전에 돈 낼 고객 1명을 확보한다. B2B는 복지관·요양원 담당자 1명과 파일럿 계약이 목표.
2. **단순한 수익 구조** — 구독(월정액) 단일 모델. 복지관 1곳 = 월 30~50만 원 기준, 20~30곳이면 목표 달성.
3. **자동화로 1인 운영** — 고객 응대·보고서 생성·일정 관리가 AI 에이전트로 자동화되어야 창업자가 영업과 제품에만 집중 가능.
4. **빠른 피드백 루프** — 기능 완성보다 실제 고객에게 보여주는 속도가 우선. 매주 최소 1명의 잠재 고객과 대화.
5. **SNS = 무료 영업팀** — 개발 과정을 공개(LinkedIn·X)해서 잠재 고객이 먼저 연락오게 만든다.

### 이 목표에서 역산한 우선순위

| 단계 | 조건 | 예상 시점 |
|---|---|---|
| 파일럿 고객 1곳 확보 | Day 36 B2B 대시보드 완성 후 데모 | Day 36~ |
| 유료 전환 1곳 | 파일럿 성과 검증 후 | Day 40~ |
| 월 300만 원 | 유료 고객 6~10곳 | V2 초기 |
| 월 1,000만 원 | 유료 고객 20~30곳 또는 엔터프라이즈 계약 | V2 중기 |

## 3. 전체 로드맵

### 🔴 1순위 — 앱 정상화 + 실사용 (즉시)

#### Day 32 — 앱 크래시 수정 + 아이콘 수정
**목표**: Play Store에서 설치한 앱이 정상 작동

**문제**:
- 회색 화면 → 프로덕션 빌드에 환경변수 없음
- 앱 아이콘이 설정한 것과 다르게 표시

**작업**:
1. EAS 프로덕션 환경변수 3개 등록 (`EXPO_PUBLIC_API_URL`, `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`)
2. 앱 아이콘 에셋 파일 확인 및 수정
3. `eas build --profile production --platform android` 재빌드
4. Play Console 새 버전 업로드

**성공 기준**: 실기기에서 앱 실행 시 로그인 화면 정상 표시

---

#### Day 33 — 삼성 AI 통화 요약 공유 실사용 테스트
**목표**: 엄마(S25) 폰으로 실제 통화 후 AI 요약 → SilverLink 공유 → 분석 결과 확인

**작업**:
1. 엄마 폰에 테스터 링크로 앱 설치
2. 통화 후 삼성 AI 요약 뜨면 → SilverLink로 공유
3. 분석 결과(위험 신호·요약) 확인
4. 버그 발생 시 수정 후 OTA 업데이트 (`eas update --channel production`)

**성공 기준**: 공유 → 분석 → 대시보드 저장까지 end-to-end 동작

### 🟠 2순위 — 컨설팅 피드백 반영 (2차 MVP)

#### Day 34 — AI 이상 징후 예측
**목표**: DB에 쌓인 통화·안전알림·점수 데이터를 AI가 분석해 "이번 주 어르신 상태 변화" 자동 감지

**AI 엔지니어 역량**: 시계열 데이터 패턴 분석, Gemini Function Calling 활용

**작업**:
1. `social_scores` + `safety_alerts` + `care_call_attempts` 4주 데이터 집계
2. Gemini에 시계열 데이터 전달 → 이상 징후 판단 (점수 급락, 미응답 연속, 위험 신호 빈도 증가)
3. 대시보드에 "AI 주의 알림" 배너 표시
4. 복지사 대시보드에 위험도 자동 업데이트 반영

**성공 기준**: 실제 데이터 기반으로 "지난주보다 점수가 15점 하락했습니다" 류의 AI 인사이트 자동 생성

---

#### Day 35 — 어르신 일정 AI 자동 생성 (수익 모델 핵심)
**목표**: 자녀가 일정을 일일이 입력하지 않아도, AI가 어르신 프로필·루틴·이전 통화 내용을 분석해 이번 주 케어 일정을 자동 제안·등록

**AI 엔지니어 역량**: AI 에이전트 자동화, RAG 기반 컨텍스트 활용

**작업**:
1. 어르신 프로필 + 최근 4주 통화 요약 + 기존 일정 패턴을 컨텍스트로 구성
2. Gemini로 "이번 주 추천 케어 일정 3~5개" 자동 생성
3. 자녀가 원클릭으로 승인 → `care_tasks`에 자동 등록
4. 주기적 자동 생성 크론 연동 (매주 월요일)

**수익 모델 연결**: 이 기능이 B2B에서 "케어매니저 업무 자동화"의 핵심 → 구독 과금 근거

**성공 기준**: 버튼 클릭 없이 이번 주 일정 초안이 자동 생성됨

#### Day 36 — B2B 대시보드 (복지관·요양원용)
**목표**: 복지사가 담당 어르신 10~50명을 한 화면에서 관리

**작업**:
1. 복지관 단위 조직 구조 추가 (복지사 1명 → 어르신 N명 관리)
2. 전체 어르신 위험도 순 목록 (기존 Day 26 확장)
3. AI 자동 이상 징후 알림 통합 (Day 34 연결)
4. 복지관 단위 월간 통계 리포트 자동 생성

**성공 기준**: 복지사 계정으로 담당 어르신 전체 상태를 3초 안에 파악 가능

### 🟡 3순위 — 디자인 개선 + AI 에이전트 자동화 (스케일업)

#### Day 38 — 웹 디자인 개선 (Claude Design + Claude Code 4단계 워크플로우)
**목표**: B2B 영업용 랜딩 페이지 + 주요 화면 UI를 프로덕션 수준으로 개편

**방법론 (YouTube 영상 기반)**:
1. **참고 디자인 수집**: Pinterest 등에서 케어·헬스테크 SaaS 레퍼런스 3개 이상 수집
2. **Claude Design으로 초안**: 레퍼런스 + 프롬프트로 디자인 생성 → Tweak(슬라이더)로 색상·폰트·여백 코드 소모 없이 조정
3. **Claude Code로 구현**: Hand-off 기능으로 디자인 → 실제 Next.js 코드 변환
4. **애니메이션 추가**: 외부 컴포넌트 라이브러리에서 효과 선택 → 특정 섹션에만 적용

**핵심 원칙**:
- Claude Design = 시각화 전담, Claude Code = 구현 전담으로 역할 분리
- 수정 시 Edit(직접)/Comment(지정)/Tweak(스타일) 3가지 방법 적절히 활용
- 프롬프트 작성 시 "유지할 디자인"과 "바꿀 부분"을 명확히 구분

**작업 대상**:
- 랜딩 페이지 (`/`) 신규 제작 — B2B 영업용 (복지관·요양원 대상)
- 대시보드 메인 화면 UX 개선
- 분석 결과 화면 (통화 요약 결과) 개선
