────────────────────────────────────────────────────────────────────────────────
  이 파일이 무엇인가
────────────────────────────────────────────────────────────────────────────────

  이 파일은 "AI 개발 도구(Claude Code)에게 건네는 작업 지시서"다.

  일반적으로 Claude Code에 "로그인 페이지 만들어줘"라고 하면 Claude는 그 한 가지만
  만들고 멈춘다. 다음에 뭘 만들지, 어떤 기술을 쓸지, 파일 구조는 어떻게 할지 — 매번
  사람이 결정하고 지시해야 한다. 그 결과 앱 하나를 만들려면 수십 번의 대화가 필요하고,
  중간에 기술 선택이 뒤죽박죽되거나 일관성이 무너지기 쉽다.

  이 파일은 그 문제를 해결한다.

  이 파일을 Claude Code에 붙여넣으면:
  ① Claude가 "어떤 기술로, 어떤 순서로, 어떤 기준으로 만들어야 하는지"를 한 번에 파악한다.
  ② 사람이 추가로 지시하지 않아도 로그인 → DB → API → 대시보드 → AI 분석 → 배포 준비까지
     스스로 순서대로 파일을 만들어 나간다.
  ③ 결과물은 실제 서비스에 올릴 수 있는 수준(프로덕션 품질)의 풀스택 웹 앱이다.

  쉽게 말해: "어떤 앱을 만들고 싶은지"만 적으면, 나머지는 Claude가 혼자 한다.

────────────────────────────────────────────────────────────────────────────────
  이 파일로 만들어지는 앱의 수준
────────────────────────────────────────────────────────────────────────────────

  이 파일은 SilverLink AI(https://silverlink-ai.vercel.app)를 만든 방식을 그대로
  추출한 것이다. SilverLink AI는 아래 기술과 기능을 갖춘 실제 서비스다:

  • 기술: Next.js 16 App Router + Supabase + Google Gemini AI + TypeScript + Tailwind
  • 인증: 이메일/비밀번호 회원가입·로그인, 세션 자동 유지
  • 데이터베이스: RLS(행 단위 보안)가 적용된 Supabase PostgreSQL
  • AI 기능: 이벤트 발생 시 AI 자동 분석 → 위험 신호 감지 → 알림 생성
  • 대시보드: 2025 Bento Grid 레이아웃, 다크 히어로 카드
  • 점수 시스템: 0~100점 지표 + 8주 추이 차트
  • 멀티 역할: 개인 모드 / 관리자 모드 (전체 현황 위험도 순 정렬)
  • 타임라인: 모든 이벤트를 시간순으로 통합해 보여주는 감사 추적
  • 모바일 대응: 375px(iPhone) + 414px(Android Chrome) 완전 지원
  • 배포: Vercel + Supabase 조합으로 무료 플랜에서 운영 가능

  도메인은 달라도 된다. HR / 교육 / 부동산 / 의료 / 법률 / 반려동물 — 어떤 분야든
  위와 동일한 수준의 앱이 만들어진다.

  사람이 직접 해야 하는 것 vs Claude가 알아서 하는 것
────────────────────────────────────────────────────────────────────────────────

  사람이 직접 해야 하는 것 (딱 3번):
  ┌──────────────────────────────────────────────────────────────────────────┐
  │  1. npm install (패키지 설치 — 터미널에서 직접 실행)                         │

  • 모든 API Route Handler
  • 대시보드, 목록, 상세, 생성 페이지
  • AI 분석 파이프라인 (이벤트 → Gemini → 결과 저장 → 알림)
  • 점수 계산, 타임라인, 관리자 대시보드
  • 모바일 반응형 UI 전체
  • .env.example, README.md

  이 파일을 사용하는 방법 (순서대로)
────────────────────────────────────────────────────────────────────────────────

  1. 아래 [앱 설명] 칸에 만들고 싶은 앱을 한국어로 자유롭게 적는다.
     (길어도 되고 짧아도 된다. "누가 · 무엇을 · 어떻게 보는가"만 있으면 충분하다.)

  2. 이 파일 전체를 복사한다.

  3. 터미널에서 새 폴더를 만들고 Claude Code를 실행한다.
     > mkdir my-app && cd my-app && claude

  4. Claude Code 첫 번째 메시지 입력창에 복사한 내용을 붙여넣고 전송한다.

  5. Claude가 파일을 만들기 시작하면, Claude가 멈추고 알려줄 때만 아래를 실행한다:
     - "npm install 해주세요" → 터미널에서 실행
     - "Supabase 프로젝트를 만들어주세요" → supabase.com에서 생성 후 키 전달
     - ".env.local을 설정해주세요" → 키 입력

  6. 완성 기준(이 파일 맨 아래 체크리스트) 항목이 전부 동작하면 끝이다.
     Vercel에 GitHub 저장소를 연결하면 바로 배포된다.

  • 이 파일 자체를 수정할 필요는 없다. [앱 설명] 칸만 바꾸면 된다.

              발주량을 자동으로 추천해주는 앱. 사장 한 명이 쓰는 단일 사용자 앱."
  • SMS·전화 발송 기능이 포함된 앱은 실제 발송이 기본 비활성화된다.
    테스트는 Mock 모드로, 실제 발송은 환경변수로 별도 활성화한다.

  import { createServerClient } from "@supabase/ssr";
  import { NextResponse, type NextRequest } from "next/server";

  export async function middleware(request: NextRequest) {
    let supabaseResponse = NextResponse.next({ request });
    const supabase = createServerClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
      {
        cookies: {
          getAll() { return request.cookies.getAll(); },
          setAll(cookiesToSet) {
            cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
            supabaseResponse = NextResponse.next({ request });
            cookiesToSet.forEach(({ name, value, options }) =>
              supabaseResponse.cookies.set(name, value, options)
            );
          },
        },
      }
    );
    await supabase.auth.getUser();
    return supabaseResponse;
  }

  export const config = {
    matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)"],
  };

  import { createServerClient } from "@supabase/ssr";
  import { cookies } from "next/headers";

  export async function createSupabaseServerClient() {
    const cookieStore = await cookies();
    return createServerClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
      {
        cookies: {
          getAll() { return cookieStore.getAll(); },
          setAll(cookiesToSet) {
            try {
              cookiesToSet.forEach(({ name, value, options }) =>
                cookieStore.set(name, value, options)
              );
            } catch {}
          },
        },
      }
    );
  }

  import { createSupabaseServerClient } from "./server";

  export async function getServerUser() {
    const supabase = await createSupabaseServerClient();
    const { data: { user } } = await supabase.auth.getUser();
    return user;
  }

  [Supabase SQL 스키마 기본 패턴]
  ──────────────────────────────────────────────
  -- 메인 엔티티 (앱 설명에 맞게 테이블명·컬럼명 변경)
  CREATE TABLE entities (
    id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id     uuid REFERENCES auth.users(id) ON DELETE CASCADE,
    name        text NOT NULL,
    metadata    jsonb DEFAULT '{}',
    created_at  timestamptz DEFAULT now()
  );
  ALTER TABLE entities ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "owner_all" ON entities FOR ALL USING (auth.uid() = user_id);

  -- 이벤트/기록 (AI 분석 결과 포함)
  CREATE TABLE events (
    id           uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    entity_id    uuid REFERENCES entities(id) ON DELETE CASCADE,
    event_type   text NOT NULL,
    payload      jsonb DEFAULT '{}',
    ai_analysis  jsonb,
    created_at   timestamptz DEFAULT now()
  );
  ALTER TABLE events ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "owner_all" ON events FOR ALL
    USING (entity_id IN (SELECT id FROM entities WHERE user_id = auth.uid()));

  -- AI 위험 알림
  CREATE TABLE alerts (
    id               uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    entity_id        uuid REFERENCES entities(id) ON DELETE CASCADE,
    event_id         uuid REFERENCES events(id) ON DELETE CASCADE,
    category         text NOT NULL,
    severity         text NOT NULL CHECK (severity IN ('high','medium','low')),
    title            text NOT NULL,
    description      text,
    suggestion       text,
    acknowledged_at  timestamptz,
    generated_at     timestamptz DEFAULT now()
  );
  ALTER TABLE alerts ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "owner_all" ON alerts FOR ALL
    USING (entity_id IN (SELECT id FROM entities WHERE user_id = auth.uid()));


────────────────────────────────────────────────────────────────────────────────
  AI 분석 파이프라인 — 모든 앱의 핵심, 반드시 구현
────────────────────────────────────────────────────────────────────────────────

  이벤트 저장 직후 → POST /api/ai/analyze 자동 호출 → 아래 5단계 실행:

  1. 해당 엔티티의 최근 10개 이벤트를 DB에서 가져와 컨텍스트 문자열로 변환
  2. Gemini에 전송:
     "당신은 [도메인] 전문가입니다. 아래 데이터에서 위험 신호를 감지하고
      JSON으로 반환하세요: { categories[], severity, summary, suggestions[] }"
  3. JSON 파싱 → events.ai_analysis 컬럼에 저장
  4. severity === "high"이면 alerts 테이블에 INSERT
  5. 대시보드 상단 배너: 미확인 alerts 건수 표시 (Suspense로 스트리밍)

  점수 계산 패턴 — 앱에 지표가 필요하면 이 공식 사용
────────────────────────────────────────────────────────────────────────────────

  score = (주요_응답률 × 0.7) + (보조_지표 × 0.3)   → 0~100점
  70점 이상: emerald (활발)
  40~69점:  amber  (보통)
